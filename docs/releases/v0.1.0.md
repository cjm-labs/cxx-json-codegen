# CJM v0.1.0 Release Notes 

CJM v0.1.0 is the first minimal working release of the CJM build-time JSON code generation pipeline.

## Highlights

- Defines the initial Metadata Model
- Generates readable C++ `to_json` and `from_json` functions 
- Provides a `cjm generate` CLI 
- Adds CMake integration through `cjm_generate(...)`
- Parses simple C++ structs 
- Interprets `json:"name"` field comments in Semantic Analysis 
- Generates `*.cjm.hpp` files under the build directory 
- Compiles and runs a basic end-to-end example 
- Adds golden tests for generated output regression coverage 

## Supported v0.1 Input Shape

```cpp 
struct User {
    std::string name; // json:"name"
    int age;          // json:"age"
};
```

## Example CMake Usage 

```cmake 
add_executable(app main.cpp)

cjm_generate(TARGET app HEADERS user.hpp)
```

## Current Backend 

The initial generated code targets:
```text 
nlohmann/json 
```

The current example uses CMake dependency wiring for nlohmann_json::nlohmann_json.

## Current Limitations 

v0.1.0 is intentionally minimal.
Not yet supported:
- full C++ parsing.
- automatic header discovery 
- nested structs 
- STL containers 
- templates 
- private fields 
- native JSON backend 
- install/package distribution 
- production-grade diagnostics for all unsupported constructs 

## Verification 

Release verification should include:

```bash 
cmake -S . -B build

ctest --test-dir build --output-on-failure 
```
