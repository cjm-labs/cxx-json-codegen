# Roadmap

## Phase 0: Project Skeleton

Goal:

```text
Make the project understandable before implementation begins.
```

Tasks:

- Add `AGENTS.md`
- Add design document
- Add roadmap
- Create initial directory layout
- Decide CMake baseline

Success criteria:

- A new contributor can understand the project goal and MVP boundary.

## Phase 1: Minimal Codegen Shape

Goal:

```text
Generate one simple `to_json` function from one annotated struct.
```

Scope:

- one input header
- one output generated header
- one annotated aggregate struct
- public scalar fields only
- nlohmann/json backend only

Example:

```text
basic.input.hpp
-> cxx-json-map
-> basic.generated.hpp
```

Success criteria:

- Generated output matches a golden file.
- The generated file compiles in an example program.

## Phase 2: Metadata Model

Goal:

```text
Separate AST discovery from code emission.
```

Introduce:

- `StructModel`
- `FieldModel`
- `TypeModel`
- `AnnotationModel`

Success criteria:

- Code emission does not directly depend on Clang AST nodes.
- Unit/golden tests can reason about generated output.

## Phase 3: Practical Field Support

Goal:

```text
Support the field types needed by benchmark/result structs.
```

Add support for:

- bool
- integer types
- floating-point types
- `std::string`
- nested supported structs
- `std::vector<T>`

Success criteria:

- A realistic benchmark result struct can generate JSON mapping code.

## Phase 4: Field Rename Annotation

Goal:

```text
Allow C++ field names and JSON keys to differ.
```

Example:

```cpp
struct [[cjm::json]] ConsumerLatency {
  [[cjm::name("consumer")]]
  std::uint64_t consumer_id;
};
```

Success criteria:

- Generated JSON key uses the annotation name.
- Default behavior still uses the C++ field name.

## Phase 5: from_json

Goal:

```text
Generate parsing code in addition to emission code.
```

This should wait until `to_json` is stable.

Success criteria:

- Generated `from_json` can parse the same shape generated by `to_json`.
- Error behavior is documented.

## Out Of Scope Until Later

Do not prioritize early:

- XML/YAML/TOML emitters
- LLVM IR passes
- Clang compiler plugins
- runtime reflection
- private field access
- inheritance
- polymorphism
- cyclic object graphs
- arbitrary template metaprogramming support
