# Website Update Prompt for CJM v0.3.0 Dogfood

Use this prompt for the AI assistant working on `cjm-labs.org`.

````md
# Task: Update cjm-labs.org with CJM v0.3.0 dogfood status

## Context

CJM is a build-time metadata compiler for Modern C++.

The current public release is CJM v0.3.0, the Practical Type Coverage release.
It generates `nlohmann/json` integration from ordinary C++ models annotated
with Go-style JSON field metadata.

CJM v0.3.0 has now been dogfooded in a real downstream CMake project:

- [lmingzhi618/ull-md-engine](https://github.com/lmingzhi618/ull-md-engine)

This validates the public downstream workflow:

```text
FetchContent
-> FetchContent_MakeAvailable(cxx_json_codegen)
-> cjm_generate(...)
-> generated *.cjm.hpp
-> normal C++ test target
```

## Goal

Update the website copy so visitors understand that CJM is now ready for early
adopters, while keeping the tone honest and early-stage.

Do not describe CJM as production-stable yet.
Do not claim full C++ grammar support.
Do not claim CJM is a JSON library.

## Messaging

Use this positioning:

```text
CJM is a build-time metadata compiler for Modern C++.
JSON is the first official backend.
```

Add or update a short early-adopter section:

```md
## Early Adopters Welcome

CJM v0.3.0 has been dogfooded in a real downstream CMake project and is ready
for early adopters who want build-time JSON integration for ordinary Modern C++
models.

Try it on a small model, report issues, and help shape the v1.0 path.
```

Mention the dogfood result:

```md
CJM v0.3.0 was validated in
[ull-md-engine](https://github.com/lmingzhi618/ull-md-engine) through the public
`FetchContent` + `cjm_generate` workflow.

The downstream tests covered optionals, vectors, ordered and unordered
string-keyed maps, nested generated structs, enums, ignored fields,
`omitempty`, fixed-width integers, and generated `to_json` / `from_json`
round trips.
```

## Links to Include

- GitHub repository: https://github.com/cjm-labs/cxx-json-codegen
- Latest release: https://github.com/cjm-labs/cxx-json-codegen/releases/tag/v0.3.0
- Downstream dogfood repo: https://github.com/lmingzhi618/ull-md-engine
- Issues: https://github.com/cjm-labs/cxx-json-codegen/issues

## UX Requirements

- Keep the homepage focused on the product, not implementation trivia.
- Make the primary CTA either "Try CJM v0.3.0" or "View on GitHub".
- Add a secondary CTA: "Report an issue".
- Make it clear that the generated file extension is `*.cjm.hpp`.
- Keep the supported-scope wording honest: practical subset, not full C++.

## Suggested Short Copy

```md
CJM v0.3.0 is ready for early adopters.

It has been dogfooded in a real downstream CMake project using the public
FetchContent + cjm_generate workflow, covering the practical JSON mapping
surface introduced in v0.3.0.
```

## Out of Scope

- Large SEO rewrite
- Blog or case-study claims
- Benchmark claims
- Community/Discord/newsletter setup
- Tree-sitter implementation details
- Any promise of production stability before v1.0
````
