# Website Update Prompt for CJM v0.3.0

Use this prompt for the AI working on the `cjm-labs.org` / GitHub Pages site.

## Prompt

Update cjm-labs.org for the CJM v0.3.0 Practical Type Coverage release.

Product positioning:
- CJM is a build-time metadata compiler for Modern C++.
- CJM is not a JSON library, reflection framework, or serialization runtime.
- JSON is the first official backend.
- The core product idea is: Standard C++ in. Standard C++ out.
- Metadata IR is the stable boundary between source understanding and backend generation.

Release message:
- v0.3.0 expands CJM's practical JSON mapping surface.
- It adds string-keyed map support:
  - std::map<std::string, T>
  - std::unordered_map<std::string, T>
  - nested supported map value types
- It adds common fixed-width numeric coverage:
  - std::int8_t / std::int16_t / std::int32_t / std::int64_t
  - std::uint8_t / std::uint16_t / std::uint32_t / std::uint64_t
- It verifies composed practical types across maps, vectors, optionals, enums,
  aliases, generated structs, and fixed-width numerics.

Suggested homepage updates:
- Update the latest release badge or release section to v0.3.0.
- Add a short "Practical Type Coverage" section.
- Keep the hero focused on CJM as a build-time metadata compiler, not a generic JSON library.
- Mention nlohmann/json as the first backend, not the product identity.
- Keep the page honest about limitations.

Suggested wording:

Headline:
CJM is a build-time metadata compiler for Modern C++.

Supporting copy:
Write ordinary C++ models with source-level JSON metadata. CJM builds a stable
Metadata IR and generates ordinary C++ integration code during your build.

Release blurb:
CJM v0.3.0 adds practical type coverage for string-keyed maps, fixed-width
numeric types, and composed model fixtures, with parser-to-generator golden
tests and generated-code compile checks.

Supported v0.3 examples:

```cpp
struct Event {
    std::uint64_t sequence; // json:"sequence"
    std::map<std::string, std::vector<std::uint64_t>> buckets; // json:"buckets"
    std::optional<std::unordered_map<std::string, std::string>> attributes; // json:"attributes,omitempty"
};
```

Limitations to keep visible:
- not full C++ grammar support
- no arbitrary JSON value yet
- no std::variant / std::any yet
- no custom converters yet
- no automatic header discovery yet
- current handwritten parser still has limited multiline declaration support
- Tree-sitter frontend research is planned for v0.3.5

Content updates:
- Add or update a release notes page for v0.3.0.
- Link to the GitHub release notes when available.
- Update any old copy that says CJM is only v0.1/v0.2 or only supports scalar fields.
- Keep metadata, Open Graph, and page descriptions aligned with:
  "Build-time metadata compiler for Modern C++."

Do not:
- Present CJM as a replacement for nlohmann/json.
- Claim full JSON Schema support.
- Claim full C++ parser support.
- Claim production package installation is finished unless the repo docs say so.
- Add SEO-heavy marketing claims.
